What I evaluate for

Capability index
FIND // GROUNDING

Grounding & hallucination

Checking whether a claim about a person, fact, or dataset is actually supported by the underlying evidence — or whether it's a plausible-sounding inference dressed up as fact.

FIND // INTEGRATION

Natural integration

Spotting the difference between context woven naturally into a response and context bolted on — the "overnarrating" that makes personalization feel robotic instead of helpful.

FIND // RANKING

Side-by-side ranking

Stack-ranking two outputs against each other on helpfulness, usability, and tone, then writing a rationale precise enough that someone else could reproduce my judgment.

FIND // BIAS

Bias & ethical risk

Flagging skewed assumptions, unfair inferences, and ambiguous edge cases before they get reinforced at scale — with structured notes engineering teams can act on.

FIND // ROOT CAUSE

Root-cause analysis

Tracing a quality failure back to where it actually started, not just where it surfaced — built from years of enterprise data audits where the symptom and the cause rarely matched.

FIND // DESIGN

Scenario & prompt design

Constructing realistic, multi-turn test scenarios from genuine personal context — the kind that actually stress-test a system instead of just confirming it works.
